# Login Feature Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Gate the HomeOS dashboard behind a single-user username/password login using a signed session cookie.

**Architecture:** Credentials are stored in the git-ignored `configs/devices.local.yaml`. When `dashboard_auth` is present, `create_app()` registers an `AuthMiddleware` that checks a signed cookie on every request (skipping `/login`, `/logout`, and `/static/*`). Session cookies are signed with `itsdangerous.URLSafeTimedSerializer` and expire after 30 days. If `dashboard_auth` is absent, the app behaves exactly as today (no auth).

**Tech Stack:** FastAPI, Starlette `BaseHTTPMiddleware`, `itsdangerous.URLSafeTimedSerializer`, PyYAML, vanilla HTML/CSS (login page reuses dashboard CSS variables).

## Global Constraints

- Python 3.10+
- No new runtime dependencies beyond `itsdangerous` (already a transitive dep; add it explicitly to `requirements.txt`)
- All new routes and middleware live inside `create_app()` — never at module scope — so tests can create isolated app instances
- Default credentials: username `admin`, password `admin` (set in `devices.local.yaml` on the Pi)
- Auth is opt-in: missing `dashboard_auth` key = no auth (existing behavior preserved)

  - `POST /login` → 303 redirect to `/` with `session` cookie on success; 303 redirect to `/login?error=1` on failure

- [ ] **Step 3: Add auth middleware inside `create_app()`**

Directly after the `_signer` initialization block from Task 1, add:

```python
    if _auth_cfg:
        class _AuthMiddleware(BaseHTTPMiddleware):
            async def dispatch(self, request: Request, call_next):
                path = request.url.path
                if path in ("/login", "/logout") or path.startswith("/static/"):
                    return await call_next(request)
                token = request.cookies.get("session")
                valid = False
                if token and _signer is not None:
                    try:
                        _signer.loads(token, max_age=_MAX_AGE)
                        valid = True
                    except (SignatureExpired, BadSignature):
                        pass
                if not valid:
                    if path.startswith("/api/"):
                        from fastapi.responses import JSONResponse
                        return JSONResponse({"error": "unauthorized"}, status_code=401)
                    return RedirectResponse(url="/login", status_code=303)
                return await call_next(request)

        app.add_middleware(_AuthMiddleware)
```

- [ ] **Step 4: Add login/logout routes inside `create_app()`**

Immediately after the middleware block (still inside the `if _auth_cfg:` block), add:

```python
        @app.get("/login")
        async def login_page() -> FileResponse:
            return FileResponse(STATIC_DIR / "login.html")

        @app.post("/login")
        async def do_login(request: Request):
            form = await request.form()
            if form.get("username") == _auth_user and form.get("password") == _auth_pass:
                assert _signer is not None
                token = _signer.dumps(_auth_user)
                resp = RedirectResponse(url="/", status_code=303)
                resp.set_cookie("session", token, max_age=_MAX_AGE, httponly=True, samesite="lax")
                return resp
            return RedirectResponse(url="/login?error=1", status_code=303)

        @app.post("/logout")
        async def do_logout():
            resp = RedirectResponse(url="/login", status_code=303)
            resp.delete_cookie("session")
            return resp
```
